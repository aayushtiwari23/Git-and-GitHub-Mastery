# Watch

## Introduction

The **Watch** feature on GitHub allows you to receive notifications about activity in a repository. It helps developers stay informed about new commits, issues, pull requests, releases, and discussions.

Unlike **Star**, which bookmarks a repository, **Watch** keeps you updated with its latest activity.

---

# What is Watch?

Watching a repository means GitHub will send notifications whenever important events occur.

These notifications can include:

- New Issues
- Pull Requests
- Commits
- Releases
- Discussions

---

# Why Use Watch?

Developers use Watch to:

- Follow open-source projects.
- Stay updated with important repositories.
- Monitor team projects.
- Track development progress.
- Receive release notifications.

---

# Notification Levels

GitHub provides different watch options.

## Participating

You receive notifications only when you're directly involved.

---

## All Activity

You receive notifications for almost every activity in the repository.

---

## Custom

Choose exactly which events you want to receive.

Examples:

- Releases
- Issues
- Pull Requests
- Discussions

---

## Ignore

Stop receiving notifications from the repository.

---

# How to Watch a Repository

### Step 1

Open any GitHub repository.

### Step 2

Click the **Watch** button.

### Step 3

Select one of the notification options:

- Participating
- All Activity
- Custom
- Ignore

---

# Watch vs Star

| Watch | Star |
|--------|------|
| Receive notifications | Bookmark repository |
| Follow project activity | Show appreciation |
| Helps track updates | Helps save useful projects |
| Notification feature | Bookmark feature |

---

# Real-World Example

You contribute to an open-source project.

Instead of checking GitHub every day, you watch the repository.

GitHub automatically notifies you whenever:

- Your Pull Request is reviewed.
- A new release is published.
- Someone comments on an issue.

---

# Best Practices

- Watch repositories you actively follow.
- Use Custom notifications for large projects.
- Ignore repositories you no longer need.
- Keep notifications organized.

---

# Common Mistakes

- Confusing Watch with Star.
- Watching too many repositories.
- Ignoring important project notifications.

---

# Interview Questions

### What is GitHub Watch?

It allows users to receive notifications about repository activity.

---

### Does Watch copy a repository?

No.

---

### What is the difference between Watch and Star?

Watch sends notifications.

Star bookmarks a repository.

---

### When should you use Watch?

When you want to stay informed about repository updates.

---

# Practice

1. Open a public repository.
2. Click **Watch**.
3. Select **Custom**.
4. Enable Release notifications.
5. Observe future notifications.

# Summary

GitHub Watch helps developers stay informed about repository activity through notifications. It is an essential feature for collaborating on projects and following active open-source repositories.
