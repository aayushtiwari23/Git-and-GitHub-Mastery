# Star

## Introduction

A Star is GitHub's way of showing appreciation for a repository. Similar to a "Like" on social media, it helps developers bookmark useful projects and indicates that a repository is valuable to the community.

Repositories with many stars often gain more visibility and trust.

---

# What is a Star?

A GitHub Star allows users to:

- Bookmark repositories
- Show appreciation
- Discover popular projects later
- Support open-source developers

Clicking the **Star** button does not copy or modify the repository.

---

# Why are Stars Important?

Stars help:

- Increase a repository's popularity
- Build the author's credibility
- Attract contributors
- Show community interest
- Improve project visibility

---

# How to Star a Repository

### Step 1

Open any GitHub repository.

### Step 2

Locate the **Star** button in the top-right corner.

### Step 3

Click **Star**.

The repository is now added to your starred repositories.

---

# How to View Starred Repositories

1. Open GitHub.
2. Click your profile picture.
3. Select **Your Stars**.

You can now access all repositories you have starred.

---

# Real-World Example

You discover a repository that teaches Git.

Instead of searching for it again later, click **Star**.

Now it is saved in your GitHub account for quick access.

---

# Star vs Fork

| Star | Fork |
|------|------|
| Bookmark a repository | Create your own copy |
| No code changes | You can modify code |
| Shows appreciation | Used for contributions |
| Takes one click | Creates a new repository |

---

# Best Practices

- Star repositories you genuinely find useful.
- Organize your learning using starred repositories.
- Explore repositories with many stars to learn best practices.
- Build repositories worth starring.

---

# Common Mistakes

- Thinking a Star copies the repository.
- Assuming Stars increase contribution count.
- Starring random repositories without using them.
