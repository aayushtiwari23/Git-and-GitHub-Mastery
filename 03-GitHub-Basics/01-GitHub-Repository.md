# GitHub Repository

## Introduction

A GitHub Repository is an online storage space where your project's code, files, documentation, and version history are hosted. It allows developers to collaborate, track changes, manage issues, review code, and share projects with others.

A GitHub repository is usually connected to a local Git repository using Git.

---

# What is a GitHub Repository?

A GitHub repository contains:

- Source Code
- README
- Commit History
- Branches
- Releases
- Issues
- Pull Requests
- Wiki (Optional)
- Actions (CI/CD)
- Security Settings

Think of it as the online home of your project.

---

# Local Repository vs GitHub Repository

| Local Repository | GitHub Repository |
|------------------|-------------------|
| Stored on your computer | Stored on GitHub's servers |
| Can work offline | Requires internet access |
| Used for development | Used for collaboration and sharing |
| Managed using Git | Managed through GitHub |

---

# Creating a GitHub Repository

### Step 1

Sign in to GitHub.

### Step 2

Click the **+** button.

### Step 3

Select **New repository**.

### Step 4

Enter a repository name.

Example:

```text
Student-Management-System
```

### Step 5

Choose:

- Public
- Private

### Step 6

(Optional)

- Add README
- Add .gitignore
- Choose License

### Step 7

Click **Create repository**.

# Public vs Private Repository

## Public Repository

- Anyone can view it.
- Great for portfolios and open-source projects.

## Private Repository

- Visible only to you and invited collaborators.
- Suitable for personal or confidential projects.

# Real-World Example

A software company creates a repository named:

```text
Online-Banking-System
```

Different teams work on:

- Authentication
- Payments
- Dashboard
- Notifications

All changes are managed through the same GitHub repository using branches and pull requests.

---

# Best Practices

- Use meaningful repository names.
- Add a professional README.
- Include a LICENSE.
- Keep the repository organized.
- Commit changes regularly.

---

# Common Mistakes

- Creating repositories with unclear names.
- Leaving the README empty.
- Uploading sensitive information such as passwords or API keys.
- Mixing multiple unrelated projects in one repository.

### What is the difference between Git and GitHub?

Git is a version control system.

GitHub is a cloud platform for hosting Git repositories.

---

### Public vs Private Repository?

Public repositories are visible to everyone.

Private repositories are visible only to authorized users.
