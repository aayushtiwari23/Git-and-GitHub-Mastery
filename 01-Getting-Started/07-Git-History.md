# History of Git

## Introduction

Git is the world's most widely used distributed version control system. It was created to solve the challenges of managing large software projects involving thousands of files and hundreds of developers.

Today, Git powers millions of repositories and is considered an essential tool for software engineers.

---

# Why Was Git Created?

In 2005, the Linux Kernel project needed a reliable version control system.

The Linux development team was using a proprietary tool called **BitKeeper**.

After licensing issues arose, they needed a new solution that was:

- Fast
- Free
- Secure
- Distributed
- Reliable

Linus Torvalds decided to build a new version control system from scratch.

That project became **Git**.

---

# Who Created Git?

Git was created by **Linus Torvalds**, the creator of the Linux operating system.

Initial development began in **April 2005**.

The first working version was completed within just a few days.

Since then, Git has evolved with contributions from developers around the world.

---

# Why is Git Called "Git"?

Linus Torvalds jokingly chose the name "Git."

In British English, "git" is an informal slang word.

Linus humorously explained that he often names projects after himself in a playful way, and the name simply stuck.

Today, the name Git is recognized globally in the software industry.

---

# Design Goals of Git

Git was designed with several important goals:

- High performance
- Distributed development
- Strong data integrity
- Non-linear development using branches
- Efficient storage
- Support for large projects

These goals still define Git today.

- Git is open source.
- Every commit has a unique SHA hash.

# Summary

Git was created in 2005 by Linus Torvalds to support Linux Kernel development. Over time, it became the global standard for version control because of its speed, reliability, flexibility, and support for collaborative software development.
