# What is Git?

## Definition

Git is a distributed version control system used to track changes in files and source code. It allows developers to save different versions of a project, collaborate with others, and restore previous versions when needed.

---

## Why was Git created?

Before Git, developers manually copied project folders like:

Project
Project_Final
Project_Final_New
Project_Final_Last
Project_Final_Final

Managing projects became difficult and error-prone.

Git solves this problem by keeping a complete history of every change.

---

## Why should you learn Git?

- Track every change made to your project
- Restore previous versions
- Work on new features without affecting the main project
- Collaborate with other developers
- Required skill for software development jobs

---

## Real World Example

Suppose you are building an ATM Management System.

Day 1

```
Login Page
```

Day 2

```
Login Page
Balance Check
```

Day 3

```
Money Transfer
```

If a bug appears on Day 3, Git allows you to return to the stable version from Day 2 without losing your project.

## Key Terms

| Term | Meaning |
|------|---------|
| Repository | Project folder tracked by Git |
| Commit | Saved version of a project |
| Branch | Separate line of development |
| Merge | Combine changes |
| Clone | Copy a repository |
| Push | Upload changes to GitHub |
| Pull | Download latest changes |

---

## Summary

Git is a version control system that records the history of a project, making development safer, faster, and easier for both individuals and teams.
