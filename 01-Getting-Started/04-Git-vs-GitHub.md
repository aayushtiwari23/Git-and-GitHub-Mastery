# Git vs GitHub

## Introduction

Many beginners think Git and GitHub are the same. They are different tools that work together.

Git is a Version Control System.

GitHub is a cloud-based platform that hosts Git repositories.

---

## Comparison

| Git | GitHub |
|------|---------|
| Version Control System | Cloud Hosting Platform |
| Tracks changes in files | Stores Git repositories online |
| Installed on a local computer | Accessible through a web browser |
| Works without an internet connection | Requires an internet connection |
| Developed by Linus Torvalds | Owned by Microsoft |

---

## Purpose

### Git

- Tracks project history
- Creates commits
- Manages branches
- Merges changes
- Restores previous versions

### GitHub

- Hosts repositories online
- Enables collaboration
- Manages Issues
- Supports Pull Requests
- Provides GitHub Actions and Releases

---

## Workflow

```
Write Code
     │
     ▼
Git
     │
Commit Changes
     │
     ▼
GitHub
     │
Share & Collaborate
```

---

## Example

Suppose you are building a Banking Application.

Git records every change you make on your computer.

When your work is complete, you push the repository to GitHub.

Your teammates pull the latest version, add new features, and push their changes back to GitHub.

---

## Advantages of Git

- Fast
- Lightweight
- Offline support
- Complete project history

---

## Advantages of GitHub

- Cloud backup
- Team collaboration
- Open-source contributions
- Portfolio for developers

---

## When to Use Git?

- Tracking local projects
- Managing versions
- Creating commits
- Working offline

---

## When to Use GitHub?

- Sharing projects
- Collaborating with teams
- Hosting open-source repositories
- Showcasing projects to recruiters

---

## Summary

Git is the tool that manages version history.

GitHub is the platform that stores Git repositories online and enables collaboration between developers.
